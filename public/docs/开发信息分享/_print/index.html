4 - Guacamole docker installation

Script of installation for Baiyun project:

//use latest version(202303)
docker pull guacamole/guacd:1.5.0
docker pull guacamole/guacamole:1.5.0


//mysql v8 encounters error: com.mysql.jdbc.exceptions.jdbc4.MySQLNonTransientConnectionException: Public Key Retrieval is not allowed
//use latest v.5.7
docker pull mysql:5.7.41



docker run --name kdic-guacd -d guacamole/guacd:1.5.0
docker run --name kdic-mysql -e MYSQL_ROOT_PASSWORD=my-secret-pw -d mysql:5.7.41


docker cp guacamole-backup.sql kdic-mysql:/
docker exec -it kdic-mysql bash
mysql -uroot -pmy-secret-pw
mysql > create database guacamole;
mysql > use guacamole;
mysql > source /guacamole-backup.sql



//if not using `MYSQL_SSL_MODE=disabled`, Guacamole will throw error. in debug mode, it said: "ssl.SSLHandshakeException: No appropriate protocol"
docker run --name kdic-guacamole \
    --link kdic-guacd:guacd         \
    --link kdic-mysql:mysql        \
        -e MYSQL_DATABASE=guacamole  \
        -e MYSQL_USER=root    \
        -e MYSQL_PASSWORD=my-secret-pw \
        -e MYSQL_SSL_MODE=disabled \
     -p 8080:8080 guacamole/guacamole:1.5.0

#list the guacamole & mysql images
docker image list

deployment

Add customization for logging page: Untar the file: guacamoleHomeBaiyun.tar to CUSTOMIZATION_FOLDER (content of CUSTOMIZATION_FOLDER will be like /extensions …)

  • use docker-compose start container
docker-compose up -d

docker-compose.yaml

#docker-compose.yaml
version: "3"

services:

  guacd:
    container_name: kdic-guacd
    image: guacamole/guacd:1.5.0
    restart: always
    network_mode: bridge
    #按需修改
    volumes:
      - ./guacd/drive:/drive:rw
      - ./guacd/record:/record:rw

  mysql:
    container_name: kdic-mysql
    image: mysql:5.7.41
    restart: always
    network_mode: bridge
    environment:
      MYSQL_ROOT_PASSWORD: 'my-Secret-pW'
    #按需修改
    volumes:
      - ./mysql:/var/lib/mysql:rw

  guacamole:
    container_name: kdic-guacamole
    privileged: true
    network_mode: bridge
    depends_on:
      - guacd
      - mysql
    environment:
      MYSQL_DATABASE: 'guacamole'
      MYSQL_USER: 'root'
      MYSQL_PASSWORD: 'my-Secret-pW'
      MYSQL_SSL_MODE: 'disabled'
      GUACAMOLE_HOME: /guaHome 
    volumes:
       - CUSTOMIZATION_FOLDER:/guaHome   
    image: guacamole/guacamole:1.5.0
    links:
      - guacd:guacd
      - mysql:mysql
    ports:
      - 8080:8080
    restart: always
  • update mysql data
docker cp guacamole.sql kdic-mysql:/
docker exec -it kdic-mysql bash
mysql -uroot -pmy-Secret-pW
mysql > create database guacamole;
mysql > use guacamole;
mysql > source /guacamole.sql

  • visit http://ip:8080/guacamole/ ， 帐号：guacadmin/guacadmin

  • change the password of user guacadmin

    • Navigate to “guacadmin -> Settings” and select the “Preferences” tab
    • In the “CHANGE PASSWORD” section, enter the current password and the new password.
    • Click “Update Password”.

docker ps

# from local PC run:
scp root@10.138.8.97:/root/guacamole/backup.sql .

5 - How to use git

How to use git to commit

  1. 确保本地代码在master分支，并拉取最新的远端代码
  • 拉取最新的远端代码 run git remote updatecmd

  • 切换到master分支 run git checkout mastercmd

  • 查看当前状态 run git statuscmd

  • 查看所有分支列表 run git branch --allcmd

  1. create your branch base on master branch

run git checkoutcmd. eg:

git checkout -b feature-use-git origin/master

  1. push your branch to origin

run git pushcmd. eg:

git push origin feature-use-git

  1. add your content

run git addcmd. eg:

git add .

  1. commit your content to local

run git commitcmd. eg:

git commit -m "how to use git"

  1. push your content to origin

run git pushcmd. eg:

git push origin feature-use-git

  1. 在coding上点击创建合并请求

  1. 添加标题， 评审者， 检查源分支和目标分支。 点击创建合并请求

  1. 评审通过后，点击合并分支进行合并

9 - 开发信息分享

背景：

为了搭建公司开发、部署测试环境，在机房搭建了两台服务器（Dell Power Edge 750，Intel Xeon Silver 4314 CPU ，128G RAM），安装了windows 2019 standard 操作系统。需在服务器上安装Docker并在其上运行Nexus和Mysql。目前服务器与internet隔离，无法上网，通过笔记本连接Xiaomi无线路由器，通过Windows远程桌面操作控制服务器 。服务器1：192.168.100.101服务器2: 192.168.100.102

问题描述：

服务器1上安装有K-spice软件，在安装Docker desktop（Version 4.15.0）并安装Nexus (Version 3.45.0)和Mysql(8.0.31)后发现如下异常现象先启动Nexus 可正常运行，如此时启动Mysql，Nexus出现异常退出，此时无log输出。先启动Mysql，再启动Nexus，无法正常启动，输出如下WARN信息。

尝试在Nexus启动后启动Nginx，未出现问题。

解决方案：

在Docker Desktop软件的Resources设置内存参数

结论：

Nexus3内置的OrientDB的未被分配足够内存资源导致。

Note:

Nexus3中的OrientDB部分关于内存资源的日志显示错误。

Docker启动程序中的参数类型需大写。
