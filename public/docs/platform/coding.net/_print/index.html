1 - Coding 中启用任务故事点和工时估算和自定义流程

背景：

  1. 在研发管理过程中需要对每个任务进行时间点估算，用以掌握迭代下的工作进度。
  2. 需定义每个任务的工时估算
  3. 在需求和任务管理流程中需要定义需求和任务的工作流为 未开始-处理中-已完成-已审核，其中前三个步骤处理人为被分配该任务的人员，如处理人接件后可按照工作进度自行更改任务的状态直至“已完成",”已审核“为该任务在变更为”已完成“后被审核人确认后更改为此状态，至此，该任务完成并审核。

操作：

  1. 使用故事点 https://coding.net/help/docs/collaboration/pattern/scrum/story-points.html#use 迭代统计 https://coding.net/help/docs/collaboration/pattern/scrum/statistics.html

  2. 任务工时估算 注：敏捷项目管理迭代状态中只能查看故事点燃尽图。

  3. 自定义工作流 https://coding.net/help/docs/collaboration/configuration/workflow.html#project

4 - coding中创建项目管理流程模板应用于项目

背景：研发项目经理提出需要在所有研发项目中增加审核流程，需要针对此需求在团队中创建项目配置方案并应用于此类研发项目中。 步骤：

  1. 团队设置中心 – 功能设置 – 项目协同 – 配置方案
  2. 配置模板工作流增加审核流程
  3. 配置模板中的事项类型（双击项目配置方案），增加预估工时属性和工时记录属性
  4. 在研发项目中使用该项目配置方案 项目协同 – 同步配置方案（将定义好的方案应用于此项目）
  5. 查看研发项目任务状态已经出现审核状态
